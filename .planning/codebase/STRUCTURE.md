## Directory Purposes

**`src/`:**

- Purpose: Extension host and webview provider implementation
- Contains: TypeScript implementation files
- Key files: `extension.ts` (primary logic), `pomodoroWebviewProvider.ts` (UI layer), `test/` (test harness)
- Build output: Compiled to `dist/extension.js` via esbuild

**`media/`:**

- Purpose: Webview presentation layer assets
- Contains: JavaScript (event handlers), CSS (theme-aware styling), SVG (icon)
- Not bundled: Loaded directly by webview provider
- CSP-protected: Inline scripts blocked; uses nonce-based script inclusion (see `pomodoroWebviewProvider.ts` line 95)

**`dist/`:**

- Purpose: Bundled extension ready for VS Code
- Generated by: `npm run compile` or `npm run package`
- Committed: Yes (to version control for distribution)
- Read-only: Generated artifact — do not edit directly

**`.vscode/`:**

- Purpose: VS Code workspace configuration for extension development
- Contains: Debug launch config, editor settings, build tasks
- For developers: Run F5 to launch extension in debug window

**`package.json`:**

- Entry point: `main` field points to `dist/extension.js`
- Activation: `onStartupFinished` event
- Commands: 5 commands registered (start, pause, reset, switch, statusBarClick)
- Settings: 4 user-configurable options (workingSessionLength, breakSessionLength, enableNotifications, autoSwitchSessions)
- Views: Activity bar container with webview view inside

## Naming Conventions

**Files:**

- Extension source: `camelCase.ts` (e.g., `pomodoroWebviewProvider.ts`)
- Test files: `{name}.test.ts` (e.g., `extension.test.ts`)
- Webview assets: `kebab-case.js` (e.g., `main.js`, `paulie-pomodoro.svg`)
- Build/config: `config.ext` or `script.js` (e.g., `esbuild.js`, `eslint.config.mjs`)

**Directories:**

- `src/` — source code
- `test/` — test files (inside `src/test/`)
- `media/` — static assets for webview
- `dist/` — distribution build output
- `.vscode/` — dev tooling (leading dot)
- `.planning/` — documentation (leading dot)

**TypeScript Classes:**

- `PascalCase` (e.g., `PomodoroWebviewProvider`)

**Functions:**

- `camelCase` (e.g., `startTimer`, `updateStatusBar`, `formatTime`)

**Constants/Enums:**

- `PascalCase` for enum (e.g., `SessionType`)
- `UPPER_SNAKE_CASE` for constants (not used in this codebase currently)

**HTML/DOM:**

- IDs: `kebab-case` (e.g., `#timer-display`, `#session-type-display`, `#start-button`)
- Classes: `kebab-case` (e.g., `.container`, `.timer-info`, `.controls`)

**Command IDs:**

- `extension-id.actionName` (e.g., `paulie-pomodoro.startTimer`, `paulie-pomodoro.statusBarClick`)

**Configuration Keys:**

- `extension-id.settingName` (e.g., `paulie-pomodoro.workingSessionLength`)

**Webview Message Types:**

- `camelCase` (e.g., `type: "updateTimer"`, `type: "startTimer"`)

## Special Directories

**`dist/` — Generated Build Artifact:**

- Purpose: Bundled extension output
- Generated by: `npm run compile` or `npm run package`
- Committed: Yes
- Do not edit: Regenerated on each build
- Includes: `extension.js` (bundled TypeScript + media references), `extension.js.map` (source map)

**`node_modules/` — Dependencies:**

- Purpose: NPM package cache
- Generated by: `npm install` or `npm ci`
- Committed: No (Git-ignored)

- Not published: Not included in VSIX
